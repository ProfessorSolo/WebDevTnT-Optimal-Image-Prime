Lesson 04 · Optimal Image Prime

Contain Yourself

One tiny CSS reset keeps your glorious hero images from blowing up your layout on small screens.

Contained hero

This image uses the global reset: img { max-width: 100%; height: auto; }

Uncontained chaos

Here we deliberately remove the reset so the image can overflow its container. Resize the viewport and watch it misbehave.

The max-width reset

Modern responsive layouts are usually fluid—containers grow and shrink with the viewport. Your images, on the other hand, ship with a fixed intrinsic width. Without guard rails, they’ll happily overflow their parents and introduce horizontal scrolling.

So we add a tiny global rule near the top of our CSS:

img {
  max-width: 100%;
  height: auto;
}
  • max-width: 100% stops images from growing wider than their container.
  • height: auto preserves the original aspect ratio.
  • Because we apply it globally (in reset.css), every image on every page starts obedient by default.

Try this in dev tools

  1. Resize the viewport down to a narrow width.
  2. Toggle the .image-frame--unbounded img rule off and on.
  3. Watch the right panel either behave… or bust out of its box.
