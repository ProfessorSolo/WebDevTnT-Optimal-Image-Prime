Lesson 04 · Optimal Image Prime

Right Pixels, Right Place

Use srcset and sizes to serve the unicorn your viewport actually needs — not the retina monster every time.

Srcset + Sizes

The browser chooses which file to download based on two things: your layout (via sizes) and the device’s pixel density.

Our unicorn ships in four intrinsic widths: 400w, 800w, 1200w, and 1792w (retina). 

<img
  src="...-large.jpg"
  srcset="
    ...-small.jpg   400w,
    ...-medium.jpg  800w,
    ...-large.jpg  1200w,
    ...-retina.jpeg 1792w
  "
  sizes="
    (min-width: 1200px) 1100px,
    (min-width: 768px) 90vw,
    100vw
  "
/>

Try this in DevTools

  1. Open DevTools → Network → check Img.
  2. Toggle responsive mode and resize the viewport.
  3. Switch DPR (device pixel ratio) to a high value to simulate a retina display.
  4. Watch which file actually downloads at each size and density.
